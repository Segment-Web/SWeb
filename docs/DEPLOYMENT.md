# Развёртывание ранней беты

## Что купить и зарегистрировать

1. Домен у любого регистратора с доступом к DNS.
2. VPS с Ubuntu 24.04, публичным IPv4, минимум 1 vCPU / 1 GB RAM / 15 GB SSD.
3. GitHub-организацию и публичный репозиторий для исходного кода.

У регистратора создать A-запись, например `chat.example.com`, на IPv4 VPS.
Открыть на сервере входящие TCP 22, 80, 443 и UDP 443. Порт 3000 наружу не
публикуется.

## Настройка

На VPS установить Docker Engine и Compose plugin, затем склонировать репозиторий.
Скопировать `.env.example` в `.env` и выставить:

```env
NODE_ENV=production
PUBLIC_URL=https://chat.example.com
ALLOWED_ORIGINS=https://chat.example.com
TRUST_PROXY=1
DOMAIN=chat.example.com
```

Запустить `docker compose up -d --build`. Caddy автоматически запросит и будет
обновлять TLS-сертификат. Проверка: `https://chat.example.com/healthz` должна
вернуть JSON с `ok: true`.

## Что эта версия уже умеет

Несколько одновременно подключённых пользователей могут общаться через один
сервер. Сервер ограничивает размер WebSocket-сообщений, частоту сообщений,
число подключений с одного IP, проверяет Origin и удаляет зависшие соединения.

## Ограничения ранней беты

- Нет аккаунтов, паролей, подтверждения email и восстановления доступа.
- Нет серверной истории и офлайн-доставки; перезапуск очищает состояние.
- Имя пользователя не является подтверждённой личностью.
- Вложения пока передаются внутри WebSocket и не подходят для большой нагрузки.
- E2EE-протокол не проходил независимый аудит.

Перед широкой регистрацией нужны аккаунты, база зашифрованных конвертов,
объектное хранилище вложений, резервные копии, мониторинг, антиспам, аудит E2EE
и юридические документы. Privacy Policy, Terms, Acceptable Use, DMCA/contact,
cookie notice и правила хранения данных намеренно отложены владельцем проекта,
но обязательны до публичного запуска с реальными пользователями.
